# INTERNET NEWS 再設計メモ

## 前提

- 既存の `ネタ系` / `2chまとめ系` / 既存カテゴリタブは維持する
- ニュース一覧ページもカテゴリ別で見られる状態を維持する
- 全面改修ではなく、`Topic中心` へ段階移行する

## 1. 現状とのギャップ

1. ホームの主役がまだ `記事一覧` 寄り
   - `index.html` ではホーム下部の `記事一覧` が面積・導線ともに強い
   - `app.js` の Topic データはあるが、見せ方が `ニュースカードの集合` に留まっている
2. Topic の説明粒度が不足
   - `whatHappened / whyHot / importantPoint / targetAudience` は一部で生成している
   - ただし一覧ページ側では十分に前面化されていない
3. 関連記事が補足扱いになっていない
   - いまは `topic.html` で記事を見せているが、ホームでは Topic の文脈より記事単体が目立ちやすい
4. カテゴリ導線が `フィルター` に寄りすぎている
   - カテゴリ別に “いま知るべき Topic” を並べる編集ビューがない
5. パフォーマンス要件の途中対応
   - lazy load と deferred render は一部ある
   - ただし「今日絶対知っとけ / ゲーム / AI を先描画、それ以外は遅延」の設計には未到達

## 2. 必要なデータ構造

既存の `trend topic` をベースに、ホーム表示用の Topic Cluster として以下を揃える。

```json
{
  "id": "switch2",
  "title": "Switch2",
  "summary": "予約開始と抽選結果が話題",
  "category": "games",
  "categories": ["games", "sns"],
  "score": 95,
  "whatHappened": "予約開始と抽選結果公開が進んだ",
  "whyHot": "複数媒体とSNSで同時に反応が広がっている",
  "importantPoint": "購入判断や需給に直結する",
  "futureOutlook": "次回抽選や在庫、追加発表が焦点",
  "targetAudience": ["ゲームユーザー", "任天堂ファン", "投資家"],
  "sourceSignals": []
}
```

今回の段階移行では以下を優先する。

- `home-topics.json`
  - `sourceSignals` を 3 件まで保持
  - `futureOutlook` を保持
- クライアント側
  - Topic から `ゲーム / AI / お得情報 / SNS・ネット / アダルト / 世の中` のホーム用セクションを生成

## 3. 必要な UI 変更

1. ホーム上段を Topic ダッシュボード化
   - `🔥 今日絶対知っとけ`
   - `🎮 ゲーム`
   - `🤖 AI`
   - 下段遅延ロードで `💰 お得情報 / 📱 SNS・ネット / 🔞 アダルト / 🌎 世の中`
2. Topic カードを強化
   - タイトル
   - 3行要約
   - `なぜ話題？`
   - `なぜ重要？`
   - `誰に関係ある？`
   - 関連記事 3 件
3. 既存の一覧は残す
   - ホーム下部のカテゴリフィルター一覧は `カテゴリ別ニュース / 話題一覧` として維持
   - `matome` など既存カテゴリは残す
4. 記事詳細の位置づけ変更
   - 記事は Topic の補足として扱う
   - 詳細ページでは引き続き `sourceSignals` を参照元として表示

## 4. 実装順序

1. ホーム文言とレイアウトを Topic 中心に変更
2. Topic カードの情報密度を上げる
3. ホーム用カテゴリセクションを追加
4. `home-topics.json` に必要項目を追加
5. 一覧ページも 4 観点カードへ寄せる
6. 次段階でサーバー側クラスタリング精度を改善する

## 今回の実装スコープ

- 既存タブを壊さずに Topic 中心レイアウトへ移行開始
- ホームの主役を Topic に変更
- 一覧ページも最低限 `何が起きた / なぜ話題 / なぜ重要 / 誰向け` を表示
- アダルト専用ページ群は現状の編集型設計を維持しつつ、ホーム導線だけ整理
